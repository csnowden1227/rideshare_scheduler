Backend Page Directory

This page lists the main HTML pages currently served by the backend so you can open them quickly, understand what each one is for, and know whether it needs a token or a location id.

Base app URL used here: https://rideshare-scheduler-axx6.onrender.com
Pages that use token or location_id need a real value to fully work.

Main App and Setup Pages

These are the pages you will use most often for onboarding, setup, practice testing, and internal access.

Protected Setup

Setup Wizard

Used to configure the client account, including maps key, CRM webhook, CRM private token, payment settings, service area, fleet, vehicle photo, license plate, year, make, model, and practice test access.

Needs: location_id and token

https://rideshare-scheduler-axx6.onrender.com/setup-wizard.html?location_id=YOUR_LOCATION_ID&token=YOUR_SETUP_WIZARD_TOKEN
Client Onboarding

Rideshare Onboarding

This is the client-facing onboarding guide that walks new SaaS users through setup steps and sends them into the setup wizard.

Works best with: location_id

https://rideshare-scheduler-axx6.onrender.com/rideshare-onboarding.html?location_id=YOUR_LOCATION_ID
Practice

Test Run

Lets the client simulate a booking, run a low-dollar practice Stripe checkout, and test the live tracking flow after setup is complete.

Needs: location_id

https://rideshare-scheduler-axx6.onrender.com/test-run.html?location_id=YOUR_LOCATION_ID
Insurance

Insurance Manager

Protected operator page for configuring an embedded insurance partner, generating booking quote shells, previewing iframe purchase flow, and storing policy state on reservations.

Needs: location_id and token

https://rideshare-scheduler-axx6.onrender.com/insurance-manager.html?location_id=YOUR_LOCATION_ID&token=YOUR_SETUP_WIZARD_TOKEN
