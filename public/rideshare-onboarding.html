CRM_One_Source Client Onboarding

Complete Your Booking System Setup

Follow each setup box in order. This guide covers Google Maps, CRM webhook setup, CRM private token creation, vehicle calendars, wizard sync, and the final workflow payload refresh step.

Open Setup Wizard
Log in to your CRM_One_Source account first. Then use the custom setup wizard link from the menu bar on the left side of your screen. Keep your setup wizard open in one tab while you gather your Maps key, webhook URL, private integration token, payment credentials, and calendar IDs.

Important Before You Begin

Your app already handles booking buffers, so you should not add extra calendar buffers inside CRM calendar setup unless you have a special business reason. The wizard is where your business profile, pricing, fleet, service area, branding, and integrations all get saved to your sub-account.

Step 1

Open the Setup Wizard From the Left Menu

  1. Log in to your CRM_One_Source account.
  2. Look at the menu bar on the left side of the screen.
  3. Find the custom menu link for the setup wizard.
  4. Click that custom menu link.
  5. Make sure the wizard opens inside your account before continuing.
Step 2

Create Your Google Cloud Project

  1. Go to Google Cloud Console.
  2. Sign in with the Google account you want to use.
  3. At the top, click the project dropdown.
  4. Click New Project.
  5. Name it something like Your Business Booking Maps.
  6. Click Create.
  7. Make sure that new project is selected.
Step 3

Enable Billing and Maps APIs

  1. In Google Cloud, click Navigation Menu.
  2. Click Billing and connect a billing account.
  3. Go to APIs & Services → Library.
  4. Enable Maps JavaScript API.
  5. Enable Places API.
  6. Enable Geocoding API.
Step 4

Create and Restrict Your Maps Key

  1. Go to APIs & Services → Credentials.
  2. Click Create Credentials → API Key.
  3. Copy the new key.
  4. Click Restrict Key.
  5. Choose HTTP referrers.
  6. Add https://app.crmonesource.com/*
  7. Add https://*.crmonesource.com/*
  8. Add https://rideshare-scheduler-axx6.onrender.com/*
  9. Click Save.
  10. Paste the key into Google Maps API Key in the wizard.
Step 5

Create Vehicle Calendars in CRM

  1. In CRM_One_Source, go to Calendars.
  2. Click Create Calendar.
  3. Create one calendar for each vehicle type.
  4. Name them clearly, such as Luxury Sedan, Standard Sedan, and Luxury SUV.
  5. Save each calendar before moving to the next one.
Step 6

Set Calendar Availability

  1. Open one of your new vehicle calendars.
  2. Go to Availability or Calendar Settings.
  3. Set the days and hours that vehicle can actually be booked.
  4. Remove days when that vehicle is unavailable.
  5. Do not add extra buffers here if your app already handles buffers.
  6. Save the availability settings.
Step 7

Copy Each Calendar ID

  1. Open one vehicle calendar.
  2. Go to the calendar details or settings page.
  3. Find the Calendar ID and copy it.
  4. Repeat this for each vehicle calendar you created.
  5. Keep those IDs ready for the wizard fleet section.
Step 8

Get Your CRM Webhook URL

  1. Go to Automation or Workflows.
  2. Open your booking workflow.
  3. Click the Webhook Trigger step.
  4. Find the generated Webhook URL and copy it.
  5. Paste it into CRM Webhook URL in the wizard.
Step 9

Create Your CRM Private Integration Token

  1. Go to Settings.
  2. Open Private Integrations.
  3. Click Create Private Integration.
  4. Name it Booking Scheduler Integration.
  5. Turn on permissions for Contacts, Calendars/Appointments, and Locations/Businesses if shown.
  6. Click Create.
  7. Copy the token immediately.
  8. Paste it into CRM Private Integration Token in the wizard.
Step 10

Add Payment Settings

  1. In the wizard, choose Stripe, Square, or CRM Invoice Only.
  2. If using Stripe, go to Stripe Dashboard → Developers → API Keys.
  3. Copy the Secret Key starting with sk_live_.
  4. Paste it into Stripe Secret Key.
  5. If using Square, copy Application ID, Access Token, and Location ID.
  6. Paste them into the matching wizard fields.
Step 11

Set Tax Rate and Service Area

  1. Enter your local Tax Rate.
  2. In the map section, search your city or main service location.
  3. Adjust the location if needed.
  4. Enter your service radius in miles.
  5. Make sure the area shown matches where you operate.
Step 12

Build Your Fleet in the Wizard

  1. Go to the Fleet or Vehicle Slots section.
  2. Click Add Vehicle or Add Row.
  3. Create one row for each vehicle calendar.
  4. For each row, enter Vehicle Type, Base Rate, Per Mile Rate, and Calendar ID.
  5. Match each vehicle row to its correct calendar name and ID.
Step 13

Click Sync

  1. Scroll to the bottom of the wizard.
  2. Click Sync.
  3. Wait for the success message.
  4. Refresh the wizard page.
  5. Confirm your settings still appear after refresh.
Step 14

Refresh the Workflow Payload and Save

  1. Go back to your booking workflow in CRM.
  2. Click the Webhook Trigger step.
  3. Click Fetch Sample Payload, Refresh Payload, or similar wording.
  4. Select the latest payload.
  5. Review the fields shown and click Save.
  6. Update your Update Contact mappings if new fields appear.
Step 15

Run a Test Booking

  1. Open your booking widget.
  2. Create one paid-in-full test booking.
  3. Confirm it hits your workflow.
  4. Confirm it lands on the correct vehicle calendar.
  5. Then run one deposit booking test if you are using deposit payments.

Important Notes

  • Create a separate CRM calendar for each vehicle type you want to schedule independently.
  • Use clear names like Luxury Sedan, Standard Sedan, and Luxury SUV.
  • Set true operating availability in CRM calendars, but do not add extra buffers there if your app already handles buffering.
  • After every important wizard sync, refresh the webhook sample payload in the workflow and click Save.

Frequently Asked Questions

Do I need a separate calendar for each vehicle?

Yes. Each vehicle type should have its own calendar so the system can check availability correctly and place confirmed bookings on the correct calendar.

Should I add buffers inside the CRM calendar setup?

Usually no. The app already has built-in buffer handling. Adding extra calendar buffers in CRM may block more time than you want and reduce availability too much.

What does the CRM private integration token do?

It gives the booking system permission to check calendars, create appointments, remove cancelled events, and sync contacts inside your CRM account.

Why do I need to refresh the webhook payload after clicking Sync?

Refreshing the webhook payload lets your CRM workflow see the latest fields and sample data structure. After selecting the latest payload, always click Save so your mappings stay current.

What should I test after setup is complete?

Run one paid-in-full test booking and one deposit test booking. Confirm the booking reaches the workflow, lands on the correct vehicle calendar, and triggers the expected reminders.

What if I get stuck during setup?

Take a screenshot of the step where you are stuck and send it to support. Include the exact step number and any error message you see.
